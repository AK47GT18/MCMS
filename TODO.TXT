## 🛠️ Setup Instructions

### 1. Copy Fixed Files

```bash
# Backup your originals first
cp docker-compose.yml docker-compose.yml.backup
cp .env.example .env.example.backup
cp docker/mysql/my.cnf docker/mysql/my.cnf.backup
cp .dockerignore .dockerignore.backup

# Now replace with fixed versions from artifacts
```

### 2. Create OnlyOffice Init Script

Create this new file:

```bash
mkdir -p docker/mysql
touch docker/mysql/init-onlyoffice-db.sql
```

Copy the SQL content from the artifact above.

### 3. Update Environment Variables

```bash
# Copy the fixed .env.example
cp .env.example .env

# Edit .env and change these passwords:
nano .env
```

**Change these values:**
- `DB_ROOT_PASSWORD`
- `DB_PASSWORD`
- `ONLYOFFICE_DB_PASSWORD`
- `REDIS_PASSWORD`
- `ONLYOFFICE_JWT_SECRET`

### 4. Build and Start

```bash
# Pull latest images
docker-compose pull

# Build PHP container
docker-compose build --no-cache php

# Start all services
docker-compose up -d

# Watch the logs
docker-compose logs -f
```

### 5. Verify Everything Works

```bash
# Check all containers are running
docker-compose ps

# Should see:
# ✅ mkaka_nginx       - running
# ✅ mkaka_php         - running
# ✅ mkaka_mysql       - running
# ✅ mkaka_redis       - running
# ✅ mkaka_onlyoffice  - running
# ✅ mkaka_phpmyadmin  - running
# ✅ mkaka_mailhog     - running
```

### 6. Test Each Service

| Service | URL | Credentials |
|---------|-----|-------------|
| **Application** | http://localhost | Your app login |
| **phpMyAdmin** | http://localhost:8081 | mkaka_user / mkaka_secure_pass_2026 |
| **OnlyOffice** | http://localhost:8080 | N/A (integrated) |
| **MailHog** | http://localhost:8025 | N/A |
| **Redis** | localhost:6379 | mkaka_redis_pass_2026 |

## 🔍 Troubleshooting

### OnlyOffice won't start?

```bash
# Check OnlyOffice logs
docker-compose logs onlyoffice

# Common issue: Database not ready
# Wait 30 seconds and restart:
docker-compose restart onlyoffice
```

### MySQL initialization issues?

```bash
# If OnlyOffice database wasn't created:
docker-compose exec mysql mysql -uroot -p

# Then manually run:
source /docker-entrypoint-initdb.d/01-onlyoffice.sql
```

### Redis connection errors?

```bash
# Test Redis connection
docker-compose exec redis redis-cli -a mkaka_redis_pass_2026 PING

# Should return: PONG
```

### Network conflicts?

```bash
# If subnet 172.25.0.0/16 conflicts, change in docker-compose.yml:
# Try: 172.26.0.0/16 or 172.27.0.0/16
```

## 🎯 Next Steps (Optional)

### Add Makefile for Convenience

Create `Makefile`:

```makefile
.PHONY: dev prod logs shell restart clean

dev:
	docker-compose up -d
	@echo "✅ Development environment started"
	@echo "🌐 App: http://localhost"
	@echo "📊 phpMyAdmin: http://localhost:8081"
	@echo "📝 OnlyOffice: http://localhost:8080"
	@echo "📧 MailHog: http://localhost:8025"

prod:
	docker-compose -f docker-compose.yml -f docker-compose.prod.yml up -d
	@echo "✅ Production environment started"

logs:
	docker-compose logs -f

shell:
	docker-compose exec php sh

restart:
	docker-compose restart

clean:
	docker-compose down -v
	@echo "⚠️  All data removed!"
```

**Usage:**
```bash
make dev    # Start development
make logs   # Watch logs
make shell  # Enter PHP container
make clean  # Destroy everything
```
